# oled-ui-astra-esp32s3

ESP32-S3 ESP-IDF port of the `oled-ui-astra` SSD1306 OLED UI demo.

The original implementation targets an STM32F103 DreamCore board. This port keeps the Astra UI code mostly platform-neutral and replaces the board layer with an ESP32-S3 HAL built on ESP-IDF GPIO, SPI, FreeRTOS, and u8g2.

Project path:

```text
D:\code\proj\edge-ai-lab\esp\oled-ui-astra-esp32s3
```

## Features

- SSD1306 128x64 OLED rendering through u8g2.
- SPI display transport on ESP32-S3.
- Two active-low GPIO keys with internal pull-ups.
- Astra launcher, menu, selector, popup, checkbox, and slider demo UI.
- FreeRTOS task entry point for ESP-IDF.

## Hardware

Default wiring is defined in `components/hal_esp32s3/hal_esp32s3.h`.

| Signal | GPIO |
| --- | --- |
| OLED MOSI | GPIO11 |
| OLED CLK | GPIO12 |
| OLED CS | GPIO10 |
| OLED DC | GPIO9 |
| OLED RST | GPIO8 |
| KEY0 | GPIO0 |
| KEY1 | GPIO1 |

The OLED uses `SPI2_HOST`. Keys are active-low and use internal pull-ups.

## Requirements

- ESP32-S3 board.
- SSD1306 128x64 SPI OLED module.
- ESP-IDF 6.1 checkout at `D:\code\proj\esp-idf`.
- Conda environment named `esp` with ESP-IDF Python dependencies.

## Build

From this directory:

```powershell
conda activate esp
. D:\code\proj\esp-idf\export.ps1
idf.py build
```

If the generated `build` directory was copied or previously configured from another project path, refresh it before building:

```powershell
idf.py fullclean
idf.py build
```

The default target is configured in `sdkconfig.defaults`:

```text
CONFIG_IDF_TARGET="esp32s3"
CONFIG_FREERTOS_HZ=1000
CONFIG_ESP_DEFAULT_CPU_FREQ_MHZ_240=y
CONFIG_SPI_MASTER_IN_IRAM=y
```

Successful builds produce:

```text
build\oled-ui-astra-esp32s3.bin
```

## Flash And Monitor

```powershell
idf.py -p COMx flash monitor
```

Replace `COMx` with the board serial port.

## Controls

- KEY0 click: move to the previous item.
- KEY1 click: move to the next item.
- KEY1 long press: open the selected page or activate the selected widget.
- KEY0 long press: close the current page or exit the active widget.
- CheckBox rows toggle immediately when activated.
- PopUp and Slider rows enter an edit overlay; use KEY0/KEY1 clicks to change the value, then long-press KEY0 to exit.

## Porting Notes

- `components/astra/` should stay close to the STM32 reference implementation.
- ESP32-S3 specific code belongs in `components/hal_esp32s3/`.
- `sdkconfig` is generated by ESP-IDF and is normally local-only. Update `sdkconfig.defaults` for committed defaults.
- The UI loop yields with a 1 ms delay to avoid starving FreeRTOS watchdog-sensitive work.
- Astra key scanning is throttled to a 10 ms cadence so the HAL long-press counter maps to roughly 1 second.
- u8g2 C sources are vendored for this port; avoid broad formatting changes in `components/hal_esp32s3/u8g2/`.
